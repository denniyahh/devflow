<objective>
Split "the agent ran validation" from "validation passed." Today `advance()` treats a
Validate stage's `DEVFLOW_RESULT: success` as passed → Ship. But an agent that runs
`/gsd-validate-phase` and *finds gaps* succeeded at its task while validation failed. This
plan adds a distinct `verdict` field (`pass` | `gaps`) to the DEVFLOW_RESULT contract, parses
it into `AgentResult`, requires the Validate prompt to emit it, and makes `advance()`'s
Validate arm treat `verdict: gaps` as a validation failure (loop back to Code / gate) rather
than advancing to Ship.

Purpose: 13b verdict-vs-ran correctness — the loop's success signal must mean "the work
passed," not "the agent exited." Runs in Wave 3: it edits agent_result.rs (Plan 03,
Wave 1), prompt.rs (Plan 02, Wave 1), and main.rs advance() (Plans 01/04, Waves 1/2), so it
sequences after all of them.
Output: a `Verdict` enum, verdict parsing, a verdict-requiring Validate prompt, and a
verdict-aware advance() Validate arm.
</objective>
