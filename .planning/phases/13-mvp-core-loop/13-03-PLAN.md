<objective>
Parse the native per-adapter completion signals that DevFlow already requests but never
reads. For Claude (`--output-format json`): read the top-level `is_error` boolean (the
authoritative error signal) and `num_turns`, treating `is_error: true` as a Layer-1 failure
independent of the DEVFLOW_RESULT marker. For Codex (`--json`): add a JSONL event-stream
parser that finds the last `turn.completed` / `turn.failed` event (the current single-JSON
parser silently returns None on Codex's per-line stream). Scope the Layer-2 commit-count
gate to Code-like stages so Define and Validate — which legitimately produce zero commits —
are no longer mis-flagged as failures.

Purpose: 13b's completion-protocol correctness. `DEVFLOW_RESULT: success` only proves the
agent's turn ended without self-reported failure; cross-checking the native envelope closes
the "verdict vs. ran" gap for every stage. This plan is agent_result.rs-only (parsing
layer), independent of the main.rs failure wiring, so it runs in Wave 1.
Output: extended Claude envelope reader, a new Codex JSONL parser, and a stage-aware Layer 2.
</objective>
