<objective>
Flip the `devflow start` default so it runs the agent inside an isolated git worktree at
`.worktrees/phase-NN/` unless an explicit `--no-worktree` opt-out is passed. Today the
default runs a `--dangerously-skip-permissions` agent directly in the primary checkout ŌĆö
the worktree is the isolation boundary that makes unattended runs safe. Update the CLI help
text for the new default and confirm the `parallel` / `sequentagent` commands (which pass an
explicit worktree bool / create their own worktrees) are unaffected.

Purpose: 13d ŌĆö unattended-safety default. This is a CLI-layer change; `worktree::add`,
`ensure_phase_worktree`, and `recover.rs`'s stale-state detection are already
worktree-path-aware. Runs in Wave 2 because it edits `crates/devflow-cli/src/main.rs`,
which Wave 1 Plan 01 also edits (different functions, same file ŌĆö sequential to avoid a
same-wave file conflict).
Output: `--no-worktree` opt-out; inverted default; updated help; regression tests.
</objective>
