Delete the confirmed-dead v1 bookkeeping from ship.rs (the `LastShip` record and the
PR-body / goal-extraction / test-summary machinery left over from the removed
`devflow confirm` / `devflow rejectpr` commands — zero non-test call sites), while keeping
the still-live cron-instructions machinery, `prepend_changelog`, and `shell_quote`. Then

issues — defining the ReviewFailed contract that Plan 01's `handle_ship_failure` consumes.

Ship stage headless-safe" — DevFlow's Rust code never creates PRs (that lives inside the
external `/gsd-ship` slash command). This plan removes the dead code and closes the
interactive-review-in-headless-mode risk (RESEARCH Pitfall 2) before the 13e dogfood run.
Output: leaner ship.rs; a Ship-stage prompt that is deterministic under
`--dangerously-skip-permissions`.
</objective>
