publish readiness against the current tree. Cargo.toml declares `license = "MIT OR Apache-2.0"`
but only `LICENSE` (MIT text) exists at the repo root — a real SPDX/dual-license mismatch. The
decision is to HONOR the dual license by adding the missing `LICENSE-APACHE` (Apache-2.0 text,
copyright holder matched to the existing LICENSE), so Cargo.toml, README's license badge/section,
and the on-disk files all agree. Then re-run the packaging dry-runs so Plan 15-05 (the actual
publish) starts from a proven-green base.

Purpose: publishing a crate that claims dual licensing without both license texts is a
correctness/compliance gap contributors and downstream users would hit on the crates.io page.
Output: a canonical LICENSE-APACHE at repo root and a clean publish dry-run.
</objective>
