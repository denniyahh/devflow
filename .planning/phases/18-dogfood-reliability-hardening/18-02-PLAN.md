<objective>
Fix WR-03 (18g): `parallel_creates_two_worktrees_and_spawns_two_monitors` waits for
two live stdout captures, runs several unrelated state assertions, then re-asserts the
same two paths still exist. A fast monitor can archive those captures between the wait
and the re-check, so the test flakes on timing rather than on behavior.

Purpose: this is the only known assertion-ordering flake left in the suite. Every
later wave of this phase runs `cargo test --workspace`; stabilizing this first means a
failure in waves 2-6 is signal, not noise.

Output: assertions relocated to run immediately after their `wait_for`, with the
now-redundant trailing re-checks removed.
</objective>
