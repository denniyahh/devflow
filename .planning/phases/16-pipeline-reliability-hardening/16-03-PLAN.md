<objective>
Make the pipeline's own completion signal trustworthy for the two Code-stage false positives that motivated this phase (both on plan 15-05, a crates.io publish with zero repo-diff success signal). Deliver two coupled fixes:

- **16a — External post-condition "Layer 0" verification.** A PLAN.md can declare a command DevFlow runs post-hoc (e.g. a crates.io sparse-index probe: `curl -sf https://index.crates.io/de/vf/devflow | tail -1 | grep -q '"vers":"1.3.0"'`). Its exit status is an authoritative signal that OUTRANKS the agent self-report (Layer 1) and commit-count fallback (Layer 2) for plans that declare it. The command source is operator-authored PLAN.md content ONLY — never agent stdout (a hard security constraint, asserted as a test).
- **16b — Retained per-stage capture history.** `cleanup_phase_files` wipes `.devflow/phase-NN-{stdout,exit}` on every stage launch, so the second false positive (an agent self-report with no evidence it attempted the work) could not be diagnosed — the capture was already gone. Rotate the outgoing stage's capture into a bounded `.devflow/history/phase-NN/` instead, keeping the last N (devflow.toml knob).

Depends on 16-01 (D-09: 16k's findings inform how the terminal signal is trusted) and 16-02 (config knobs).
Purpose: no external-only plan can be marked "success" on self-report alone, and every false positive is now forensically recoverable.
Output: `verify.rs` Layer-0 runner, `evaluate_agent_result` Layer-0 prepend, `archive_phase_files` rotation, tests.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Agent stdout → DevFlow completion evaluation | Untrusted: the agent can self-report success it did not achieve (the exact incident) |
| PLAN.md frontmatter (operator-authored) → verify.rs command | Trusted input, same level as GSD command construction; the ONLY allowed source of the verification command |
| Archived capture files under .devflow/history/ | May contain agent session data (session_id/cost) — must stay gitignored (16i, plan 16-05) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-16-09 | Spoofing | Agent DEVFLOW_RESULT: success with no work done, accepted as authoritative | high | mitigate | Layer 0 external probe OUTRANKS Layer 1 self-report for declared plans (Task 2) |
| T-16-10 | Tampering / injection | Verification command sourced from agent-controlled stdout → arbitrary command execution | high | mitigate | Command read ONLY from PLAN.md frontmatter; enforced by test, not convention (Task 2) |
| T-16-11 | Information disclosure | Retained capture history leaking session telemetry into git | high | mitigate | .devflow/history/ path derived from history_dir(); gitignore coverage enforced by 16i invariant test (plan 16-05) |
| T-16-12 | DoS | Unbounded capture history filling disk | medium | mitigate | prune_history bounds generations by capture_retention (Task 1) |
| T-16-13 | Tampering | Shell metacharacters in a legitimate PLAN.md command | low | accept | Operator-authored, trusted; run via `sh -c` intentionally to allow pipes (crates.io probe uses a pipe) — same trust model as DEVFLOW_GATE_NOTIFY_CMD |
