<objective>
Turn two classes of Phase 15 dogfood incident into deterministic, cheap `#[test]` invariants (D-05: cargo tests, no new hook machinery — they run locally, in CI, and inside the Code stage's own test run so an agent cannot self-report success past a failing invariant):

- **16i — `.gitignore`/runtime-file invariant.** The leaked-telemetry incident (`.devflow/phase-07-stdout` with session_id/cost data sat committed) happened because `.gitignore` was not updated after the 14a per-phase rename. A test enumerates every `.devflow/`-writing path from source path-constructor functions (NOT a hardcoded list — RESEARCH Pitfall 4) and asserts `.gitignore` covers each, so the next rename-without-gitignore-update fails at commit/CI time.
- **16c — Deterministic doc-claim checker (D-06, D-07, D-08).** A grep-and-cross-reference test (not an LLM judgment) with two layers: (1) existence — every env var / CLI flag / DEVFLOW_* knob / file path named in operator docs must exist in source; (2) pinned claims — a hand-maintained set of `(doc claim, source assertion)` pairs for specific value/behavior claims (existence-only would NOT have caught the RUST_LOG-default incident). Bidirectional (D-08): docs→source AND source→docs coverage for env vars/subcommands. Scoped scan + allowlist (D-07): only README/ARCHITECTURE/CONTRIBUTING/OPERATIONS/docs; skip CHANGELOG and .planning; a checked-in allowlist with required reasons.

Depends on 16-02 (reuse the toml-loading idiom for the allowlist file — do not invent a second parser).
Purpose: the two regression classes that cost the Phase 15 run become mechanical guards that are too fast to be worth skipping (operator's minimal-live-test principle).
Output: `doc_check.rs` (all `#[test]`), a checked-in allowlist, and pub(crate) exposure of the few private path constructors 16i must enumerate.
</objective>

| Boundary | Description |
|----------|-------------|
| Operator docs (README/ARCHITECTURE/…) → checker | The checker READS docs and greps source; it must never execute anything it finds in docs (no eval) |
| Source path constructors → gitignore assertion | The set of .devflow/-writing paths is the security-relevant surface for telemetry leakage |

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |

| T-16-18 | Information disclosure | A new .devflow/-writing path added without gitignore coverage leaking session telemetry into git | high | mitigate | 16i source-derived invariant test asserts coverage for every constructor-derived path (Task 1) |
| T-16-19 | Tampering / injection | Doc-checker executing content it scans (eval-like) | medium | mitigate | Checker only greps/compares strings; V5 constraint — no code path executes doc content (Task 2) |
