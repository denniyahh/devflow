<objective>
Replace the single generalist, standard-depth `/gsd-code-review` at Ship — proven to miss distinct findings four times in the Phase 15 dogfood run — with a deep, multi-angle review, and add a lighter incremental review so drift is caught earlier.

- **16d — Deep + multi-angle Ship review (D-01, D-02).** One shared Ship prompt (harness-agnostic per D-01) carries the four incident-derived angles — doc-accuracy cross-reference, security/leaked-data, CI/build correctness, external-state claims — plus one generalist deep pass, with a capability-conditional instruction: dispatch one finder subagent per angle in parallel where the harness supports it (Claude Code), otherwise run each angle as a focused sequential pass; merge all findings into one deduplicated REVIEW.md. This mirrors this project's own proven Phase 14 post-ship review (8 finder angles → dedup → verify). The angle list is config-extensible (D-02, devflow.toml) and snapshot-tested.
- **16e — Incremental per-plan/per-wave review (D-04, planner's discretion).** Run a lighter review as each plan/wave lands, not only once at whole-phase Ship, so drift does not compound across waves. Design decision (recorded below): implement 16e as a self-review instruction in the Code-stage prompt rather than a new hook subsystem — keeping it harness-agnostic and within prompt composition (D-05's spirit: no new machinery in a hardening phase). The Ship review remains the authoritative gate; the incremental pass is advisory recall insurance.

Depends on 16-02 (angle-override config knob).
Purpose: Ship's gating review catches in one pass what four single-pass reviews caught one at a time.
Output: extended `ship_stage_prompt` + Code-stage incremental instruction + snapshot tests, all in prompt.rs.
</objective>
