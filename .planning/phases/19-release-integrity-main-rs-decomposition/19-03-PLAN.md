<objective>
Close 19b: `Git::commit_path` (`crates/devflow-core/src/git.rs:325`) forces a
commit even when the scoped path has nothing to commit. `hooks::version_bump`
then tags that commit, so a release tag can name a version sitting on a commit
that contains nothing. The function's own doc comment already claims the
correct contract; the implementation contradicts it, and the
`nothing to commit` match arm immediately below is unreachable dead code.

Fix per D-16: stop forcing the commit and let the existing `nothing to commit`
arm become the genuine no-op it was written to be.

Per D-17, also determine and **record** whether `commit_all`'s empty-commit
behavior is load-bearing anywhere — without changing `commit_all`.

Purpose: release-record integrity. A tag that names a commit with no content
misrepresents what was released.
Output: one removed argument, an honest doc comment, two new tests, and a
written finding about `commit_all`.
</objective>
