<objective>
Lay the foundation for the `main.rs` split (19c–19f) without moving any
production code yet.

Three things must be true before the first cluster moves, and none of them are
incidental details of a larger move task:

1. A recorded baseline exists. `19-VALIDATION.md`'s equivalence proof compares
   the test name set and pass counts before and after; "before" has to be
   captured and committed while it is still trivially obtainable.
2. `CliError` is `pub(crate)`. `19-RESEARCH.md` Pitfall 6 verified that **zero**
   items in `main.rs`'s production half carry any visibility keyword today.
   `CliError` is returned by nearly every function in the file, so it blocks
   compiling any moved cluster.
3. The shared `ENV_MUTEX` is hoisted into a module every future sibling can
   import (D-01), with the D-04 invariant written down beside it.

Purpose: this is the plan where the phase's principal risk is actually
exercised. If a single shared mutex cannot preserve today's serialization, that
finding surfaces here — before four cluster moves have been built on top of it.
Output: a baseline artifact, a visibility pass, and `test_support.rs`.
</objective>
