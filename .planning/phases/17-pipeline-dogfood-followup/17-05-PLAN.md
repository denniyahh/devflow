<objective>
Add the two pre-launch guards that stop wasted agent time and stale-binary confusion. A generic preflight (plus an adapter hook) runs before every stage launch and fails as a named gate — not a hard exit — when the stage's actual requirements (interactivity, security artifact, credential validity, reviewer set) are unmet. And the workflow_started event records build provenance, with a self-dogfood staleness check that blocks a known-stale DevFlow-on-itself run while only warning ordinary projects.

Purpose: The Phase 16 dogfood surfaced Ship's empty reviewer set and invalid GitHub auth only after Ship's work had run, and read false evidence from a stale binary for an entire spike. Both are readiness failures that must be caught before an agent spawns.
Output: AgentAdapter::preflight default method; run_preflight in launch_stage; staleness check; D-21 provenance payload.
</objective>

| gh auth status output -> events.jsonl / gate context | external command output; must not leak token/account material into logs |
