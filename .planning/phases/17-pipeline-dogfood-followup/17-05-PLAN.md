<objective>
Add the two pre-launch guards that stop wasted agent time and stale-binary confusion. A generic preflight (plus an adapter hook) runs before every stage launch and fails as a named gate — not a hard exit — when the stage's actual requirements (interactivity, security artifact, credential validity, reviewer set) are unmet. And the workflow_started event records build provenance, with a self-dogfood staleness check that blocks a known-stale DevFlow-on-itself run while only warning ordinary projects.

Purpose: The Phase 16 dogfood surfaced Ship's empty reviewer set and invalid GitHub auth only after Ship's work had run, and read false evidence from a stale binary for an entire spike. Both are readiness failures that must be caught before an agent spawns.
Output: AgentAdapter::preflight default method; run_preflight in launch_stage; staleness check; D-21 provenance payload.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| gh auth status output -> events.jsonl / gate context | external command output; must not leak token/account material into logs |
| embedded provenance + target git checkout -> block/warn decision | a stale binary is untrusted evidence about source; the decision gates or warns based on workspace identity |
| adapter preflight reason -> gate context | adapter-supplied reason string flows to a notification |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-17-12 | Spoofing | stale-binary self-dogfood | high | mitigate | D-17 workspace-identity detection + D-19 composite staleness block a known-stale DevFlow-on-itself run before it can produce false evidence (the Phase 16 root cause) |
| T-17-13 | Information Disclosure | gh auth credential probe | medium | mitigate | Only a boolean pass/fail + short reason is recorded; raw gh stdout/stderr is never written to events.jsonl or a gate context |
| T-17-14 | Denial of Service | preflight hard-exit on unattended run | high | mitigate | D-15 named gate + notify (WR-11 idiom), never a hard exit to stdout invisible to a cron-launched run; indeterminate staleness warns rather than blocking (Pitfall 4) |
| T-17-15 | Tampering | staleness git subprocess | low | mitigate | argv-array git shelling, read-only; exit-code contract handled per Pitfall 4 so a shallow/unknown-commit case is indeterminate not a false block |
| T-17-SC | Tampering | dependency supply chain | low | accept | No packages installed; D-20 forbids new deps |
