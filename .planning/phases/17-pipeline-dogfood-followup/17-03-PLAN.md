<objective>
Rework the completion cascade so no ambiguous outcome can silently advance and a legitimately external-only stage can complete cleanly. Split Layer 3's blanket Unknown so that "process gone, nothing accounted for" (zero commits, no declaration) becomes a Failed outcome that notifies a human, while "process gone but commits exist" stays Unknown (gated downstream). Extend Layer 0 to run for every stage and to count an approved, all-passing external post-condition as affirmative Success — without relaxing the approval-mismatch security property.

Purpose: This is the load-bearing half of 17a. D-03's three-way distinction ("no code produced" vs "nothing accounted for what was done") only works if Layer 0 can vouch for a declared external deliverable and Layer 3 stops treating an unaccounted-for vanished agent as advanceable.
Output: Reworked evaluate_layer0 and evaluate_layer3 in agent_result.rs with unit tests.
</objective>
