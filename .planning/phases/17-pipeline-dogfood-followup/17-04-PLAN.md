<objective>
Make the stage-advance dispatch fail-closed and outcome-typed. Replace advance()'s two-value failed boolean with an exhaustive match on outcome_policy::decide_action so no outcome (least of all Unknown) can silently advance. Wire the primary monitor loop's RateLimited branch into the cron-instructions auto-resume machinery it never called before, route infrastructure outcomes through the separate counter, and replace reason:null success events with a structured evidence record.

Purpose: This is where 17a (never-advance) and 17b (typed outcomes, retry policy, evidence) converge on the exact code path every ordinary `devflow start` run uses and from which the Phase 16 dogfood findings came. It is the most fragile change in the phase — keep the diff small and every changed line traceable to a decision.
Output: Reworked advance() dispatch, primary-loop rate-limit resume, infra-counter gating, structured advance_evaluated evidence.
</objective>
