<objective>
Fix WR-01: `monitor::spawn_monitor` builds a shell script string and interpolates the agent command into it via `shell_escape` (which only escapes single quotes). The agent `program`/`args` are the one externally-influenced part of that script and are the primary command-construction/injection surface. Reviewer-assessed low risk (prompt is internal, `--agent` is enum-constrained), but fragile.

Purpose: Eliminate the injection surface entirely by delivering the agent command to the monitor as separate argv rather than as interpolated script text, while preserving the detached-monitor architecture (own the agent, capture pid/stdout/stderr/exit, then run `devflow advance`).
Output: `spawn_monitor` rewritten so the untrusted agent program+args are passed as shell positional parameters (referenced via `"$@"`) — or spawned directly as an argv vector — with `shell_escape` no longer applied to `program`/`args`.
</objective>

  <action>
    Restructure `spawn_monitor` so the agent `program` and each `arg` are delivered to the detached monitor as shell positional parameters instead of being interpolated into the script body via `shell_escape`. Invoke the wrapper as `Command::new("sh").arg("-c").arg(SCRIPT).arg("sh").arg(program).args(args)` so that inside SCRIPT the agent command is `"$@"` (after positioning the internal file paths). Keep the orchestration plumbing the script already performs: trap TERM/INT, `cd` into the workdir, launch the agent in the background with stdout/stderr redirected to the capture files, record `$!` to the pid file, `wait`, write `$?` to the exit file, then run the devflow binary's hidden `advance <project_root>`. The devflow-derived paths (workdir, stdout/stderr/exit/pid files, binary, project_root) are internal, not attacker-controlled — they may remain interpolated (keep `shell_escape` for those), but `program`/`args` must NOT pass through `shell_escape`. If cleaner, deliver the internal paths as additional positional parameters too. Remove the `agent_cmd`/`shell_escape(program)` interpolation for the agent command. Do not change the function signature or the returned monitor pid semantics.
  </action>
