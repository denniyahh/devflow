<objective>
Fix WR-07: `workflow::save_state` writes `state.json` with a plain `std::fs::write` (truncate-then-write, non-atomic). A process kill mid-write leaves an empty or partial JSON file; the next `load_state` fails to parse and the workflow is permanently stuck — the single highest-severity reliability defect for an unattended "walk away" tool.

Purpose: Guarantee `state.json` is only ever replaced atomically, so an interrupted save cannot corrupt persisted workflow state.
Output: `save_state` rewritten to write a sibling temp file then rename over the target, plus a test proving the write path.
</objective>
