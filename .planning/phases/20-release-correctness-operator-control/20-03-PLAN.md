<objective>
20c ‚Äî Plan-only pipeline mode (`devflow start --until <stage>`).

`devflow start` always runs the full Define‚ÜíPlan‚ÜíCode‚ÜíValidate‚ÜíShip pipeline;
`--mode supervise` only changes where it gates. There is no way to stop after
Plan except killing the monitor, which strands state and orphans a worktree ‚Äî
the exact mess 18a/18b exist to clean up. This blocks cheap, frequent dogfood
runs (the project's highest-yield bug source).

Research (Pattern 3) found the single clean interception point:
`pipeline_gate::transition` (pipeline_gate.rs:51) ‚Äî every meaningful stage
advance funnels through it (including all three Validate‚ÜíShip branches in
`handle_validate_outcome`). Ship itself never calls `transition`
(`handle_ship_outcome` calls `finish_workflow` directly), so `--until ship` is a
semantic no-op that must be rejected (D-07). `loop_back_to_code`
(pipeline_gate.rs:84) bypasses `transition` and must stay untouched.

**The doctor gap this plan MUST close (D-09 + review Codex HIGH):**
`reconcile_phase` (commands.rs:1329) runs BOTH `check_dead_agent`
(commands.rs:1247-1261) and `check_dead_monitor` (commands.rs:1270-1284).
`check_dead_agent` fires `Severity::Problem` for any `is_agent_stage()` phase
with a dead agent pid; `check_dead_monitor` fires `Problem` when `liveness()` is
`Stuck` (dead recorded monitor). A `--until plan`-stopped phase sits at
`Stage::Plan` with a dead agent pid (and potentially a stale `monitor_pid`) on
disk, so it would be misreported as crashed unless BOTH checks recognize the
stop marker. Task 1's stop path therefore also clears `monitor_pid`/`gate_pending`,
and Task 2 adds a `facts.stopped ||` guard to BOTH checks. Without this the
"clean stop point" goal is not met ‚Äî it is an explicit task (Task 2) with its
own acceptance criterion.

Purpose: a terminal-but-not-failed stop short of Ship, invisible to doctor as a
problem, leaving no polling monitor behind.
Output: `--until` flag; new `State` stop fields; `transition` interception;
`check_dead_agent` + `check_dead_monitor` stop-awareness; `resume` stop-clearing;
regenerated help snapshot + OPERATIONS.md.
</objective>
