<objective>
Deliver unit 21c (D-06): give `sequentagent`'s second agent a tracked record so it is observable while it runs. Today one `phase-N-agent-pid` file is written per phase, keyed only by `(project_root, phase)` with no agent-slot dimension; `sequentagent` calls `run_agent_blocking` twice (agent A, then agent B) writing the same paths, and because it never persists a `State`, `devflow status` shows nothing for either agent. The monitor half of the original item shipped in v1.5.0 (18b); this is narrowed to the sequentagent second-process gap.

Re-scope finding (D-06's mandated first step, stated not assumed): status/doctor derive phase visibility from `workflow::list_states()`, and sequentagent persists no state, so BOTH agent A and agent B are currently invisible. This plan takes the **narrow** reading D-06's text calls for â€” give the second process a distinctly-labeled record (slot A vs B + AgentKind) and surface it in `status` â€” while explicitly recording that the record labels whichever agent is currently running (so agent A is covered too, incidentally). Surfacing is scoped to `status` (the live-observability command); `doctor` integration is deliberately NOT included here â€” doctor reconciles persisted `State`, and adding a text-only line without a matching `--json` key would reintroduce the human/json inconsistency 18-WR01 fixed. This is an explicit planner decision (assumption-delta), reversible.

This plan is Wave 4; it shares `commands.rs` with 21-02/21-03, hence last.

Purpose: make the sequentagent handoff legible instead of a black box.
Output: a slot record + a `status` section that names the running agent.
</objective>
