<objective>
Deliver unit 21a (D-03): purely additive operator discoverability across four gaps — (1) `devflow gate show <phase>` to read a truncated gate reason in full, (2) surface the rate-limit reset time (already computed, buried in the cron-instructions file) in human `status` output, (3) an in-stage progress line in `status`, and (4) recovery-verb hints from a stuck state. No behavioral or correctness change to the pipeline — every task is presentation of data the codebase already computes.

Scope/assumptions (surfaced, not assumed silently): `gate show`'s flag surface is **positional `phase` + `--stage`**, matching `gate approve`/`gate reject`'s existing convention (RESEARCH Open Q3 recommendation) rather than `gate list`'s no-arg shape. `status` progress is represented as **elapsed time in the current stage** (no fixed stage-count denominator exists to render `N of M` reliably). This plan is Wave 2 and depends on 21-01 landing (D-07 sequencing).

Purpose: make devflow's own live state legible from a plain terminal.
Output: a new `gate show` subcommand + richer, still read-only `status` output.
</objective>
