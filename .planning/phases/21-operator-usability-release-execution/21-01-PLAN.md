<objective>
Make `embedded_commit_is_stale`'s strict-ancestor arm content-aware (unit 21d, D-07). When the running `devflow` binary's embedded commit is a strict ancestor of HEAD, run `git diff --name-only <embedded> HEAD` and filter the changed paths through the EXISTING `affects_compiled_binary` predicate: return `Fresh` when no build input changed (the docs-only case), `Stale` only when one did. This retires the false-positive hard-block observed live during this very phase's launch (binary at `7163347`, worktree HEAD `3a17381`, delta = `.planning/*` only, yet hard-blocked) and the `[[feedback-dogfood-rebuild-before-revalidate]]` workaround.

Sequence context: this plan is Wave 1 and runs FIRST (D-07 sequencing) — the guard hard-blocks Phase 21's own Plan/Code/Validate stages after every `.planning/` commit, so narrowing it first removes the no-op-rebuild tax for the rest of the phase. It is the only Wave-1 plan; every other plan (21-02..21-04) depends on it landing first. Scope fences: 999.25/999.28 are out of this phase (D-01); the phase theme is reversible/detection-only legibility (D-02); 21e changelog content stays excluded stretch (D-08).

Purpose: DevFlow's own primary workflow commits docs constantly; a docs-only commit must not re-arm a hard block after every build.
Output: A narrowed ancestry arm + block-message fix, plus regression coverage that proves the real-change protection still holds.
</objective>
